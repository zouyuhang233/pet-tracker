# 定位器通信协议 (STM32 → TCP服务器)

## 概述
STM32 通过 MN316 NB-IoT 模块以 TCP 连接向服务器发送数据。
服务器监听端口 8080，接收数据后解析并存储。

## TCP 连接信息
- 服务器 IP: 8.134.127.141
- 服务器端口: 8080
- 协议: TCP (STREAM)

## STM32 端 AT 命令序列
```
AT+NSOCR="STREAM",6,6008,2      // 创建 Socket
AT+NSOCO=0,8.134.127.141,8080   // 连接服务器
AT+NSOCFG=0,0,2                 // 配置数据格式（普通字符串）
AT+NSOSD=0,length,data           // 发送数据
```

## 数据格式
### 行格式
每条数据以换行符 `\r\n` 结尾，一行一个数据项。

### 支持的指令类型

#### 1. GPS 数据 (NMEA 原始格式)
直接发送 NMEA 0183 格式的 GPS 语句（以 `$` 开头）：
```
$GNGGA,014845.00,2233.1234,N,11357.5678,E,1,08,1.2,45.6,M,0.0,M,,*6A\r\n
$GPTXT,01,01,01,"Antenna OK"*34\r\n
```

#### 2. 步数数据
```
STEP:42\r\n
```

#### 3. 状态信息
```
STATUS:Battery=85%,Signal=23,RSSI=-67\r\n
```

#### 4. 自定义数据
```
ACC:0.15,0.22,0.98\r\n
```

## 示例发送流程
1. 初始化 MN316 模块
2. 创建 Socket 并连接服务器
3. 定时发送数据（建议 5 秒一次）：
   ```
   $GNGGA,014845.00,2233.1234,N,11357.5678,E,1,08,1.2,45.6,M,0.0,M,,*6A\r\n
   STEP:42\r\n
   ```
4. 保持连接，断线自动重连

## 服务器响应
- 服务器收到数据后不会主动回复（单向通信）
- 服务器会将数据广播给所有连接的 Web 客户端
- 如果服务器发送数据给设备，设备需通过 `AT+NSORF` 查询接收

## 数据解析
服务器端会自动解析：
- `$GNGGA` 语句 → 提取经纬度、海拔、卫星数
- `STEP:` 行 → 提取步数
- `STATUS:` 行 → 提取设备状态
- 其他行 → 作为原始日志存储

## 注意事项
1. MN316 发送数据时使用 `AT+NSOSD` 命令
2. 数据长度不要超过 512 字节
3. 建议每 3-5 秒发送一次心跳数据
4. 设备断线后需重新建立 TCP 连接
